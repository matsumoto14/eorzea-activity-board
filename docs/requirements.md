# eorzea-activity-board 要件定義

## 背景・目的

身内の FF14 Discord グループ向けに、日々の活動(レベルレ・モブハン・PW作成・地図など)を
ゆるくマッチングする低コスト Bot を作る。

最初に実現したい価値:
**「Bot が 20 時に自然な遊びの候補を出し、メンバーがボタンを押すだけで PT 相談が始まる」**

## 方針

- 公開サービスではなく身内向けの小規模 Bot
- スケール・多機能性よりも **低コスト・低運用負荷・シンプルさ** を優先
- 無料枠または低額で運用できる構成
- 実装言語は Go

## ユーザー体験の原則

- ユーザーは募集文を書かない
- ユーザーは Slash Command を覚えない
- 操作は **固定メッセージ・ボタン・セレクトメニュー・モーダル** のみ
- 20 時の投稿では「参加する」「人数足りたら呼んで」「今日は無理」などのボタンを押すだけ
- 人数が集まったら Bot が自動でスレッドを作成し、そこで PT 相談が始まる

## 機能要件(MVP)

| # | 機能 | 概要 |
|---|------|------|
| 1 | 固定メッセージ型の入口 | 設定用チャンネルに Bot がピン留めメッセージを設置。ボタンから各操作へ |
| 2 | やりたいこと設定 | 普段やりたい活動(複数)・参加しやすい時間帯(複数)・参加スタンスをセレクトメニューで設定 |
| 3 | 今日の状態変更 | 「今日いける」「呼ばれたら」「今日は無理」をボタンで切り替え |
| 4 | 武器作成の簡易進捗管理 | モーダルで自分の進捗メモ(PW・RW など)を更新。一覧をボタンで閲覧 |
| 5 | 20 時の自動投稿 | 毎日 20:00 JST に「今日成立しそうな活動」の候補を投稿 |
| 6 | 参加ボタン | 候補ごとに「参加する」「人数足りたら呼んで」「今日は無理」 |
| 7 | スレッド自動作成 | 「参加する」が活動ごとの成立人数に達したら自動でスレッド作成・参加者へ通知 |
| 8 | フリー募集 | カテゴリに収まらない内容を自由テキスト(60 字まで)で入力し、その場で募集カードを立てる |

### 対象アクティビティ(2 階層: カテゴリ + 種目)

セレクトメニューの仕様上の上限(25 件)と、身内の人数で票が割れない粒度を両立するため、
アクティビティは **カテゴリ(マッチングの単位)+ 種目(募集時に選ぶ詳細)** の 2 階層とする。
やりたいこと設定・20 時サマリー・候補人数はカテゴリ単位。種目は募集を開くときに
1 ステップだけ選び(「相談して決める」も可)、募集カードとスレッド名に載せる。
募集は **種目ごと** に開ける(同日にレベルレとエキルレの募集が並立できる)。
カテゴリ・種目のどれにも収まらない内容は、フリー募集(自由テキスト)で立てる。

成立人数は全カテゴリ 2 名(2 人集まれば即スレッドを立て、残りはそこで誘う運用)。

しきい値・カテゴリ・種目はコード内定義(`internal/activity/activity.go`)で変更可能。
カテゴリは 25 件、種目は「相談して決める」を足して 25 件が上限。

旧アクティビティ ID(levelre / exroulette / alliance / pw / goldsaucer)は読み出し時に
現行カテゴリへ自動解決され、メンバーの再登録は不要。

### Bot のペルソナ

ユーザー向けの文言はヒルディブランド調(「ごきげんよう、諸君!」「〜であります」)で統一する。
ボタンやセレクトのラベルは操作性優先で簡潔に保ち、口調は説明文・応答メッセージに乗せる。

## MVP 対象外(明示的にやらないこと)

- Web UI
- Lodestone 連携
- 高度なスコアリング・最適マッチング
- DM 通知
- 週次サマリー
- 外部公開・マルチギルド対応

## 非機能要件

- **コスト**: 無料枠または月数百円以内(最小 VPS / コンテナホスティングの最小構成)
- **運用負荷**: 単一プロセス・単一ファイル DB(SQLite)。バックアップは DB ファイルのコピーのみ
- **規模**: 1 ギルド・メンバー数十人を想定。性能チューニング不要
- **可用性**: 落ちても困らない(再起動すれば復帰)。ベストエフォート

## 運用(管理者のみ)

- `!eab setup` : 実行したチャンネルを掲示板チャンネルとして登録し、入口の固定メッセージを設置・ピン留め
- `!eab post` : 20 時を待たず今日の候補を手動投稿(テスト・臨時用)
- `!eab help` : コマンド一覧
